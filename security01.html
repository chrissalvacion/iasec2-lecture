1. Least Privilege

Definition: Users and systems should only be given the minimum access required to perform their tasks.

By limiting privileges, the potential damage caused by mistakes or attacks is drastically reduced.

Example: A student in a school system can view their own grades but cannot edit them. Only authorized faculty can update records.

2. Defense in Depth

Definition: Using multiple layers of security controls. This ensures that if one layer fails, others still provide protection.

This approach recognizes that no single security measure is perfect.

Example:
1. Login Password
2. Two-Factor Authentication (2FA)
3. Firewall
4. Data Encryption

3. Separation of Duties

Definition: Critical tasks are divided among multiple individuals or systems.

This prevents a single person from having enough power to commit fraud or abuse without detection.

Example: The employee who requests a purchase is different from the manager who approves the payment.

4. Secure by Design / Fail-Safe

Definition: Systems should be secure by default. When errors occur, the system should deny access (Fail Closed) rather than allow it (Fail Open).

Example: An electronic door lock loses power.
Fail-Safe: The door remains locked to keep intruders out.
(Note: In physical safety scenarios, doors might unlock for fire safety, but in digital security, we default to deny).

Part II: Security Models

Formal rules and structures that guide how security policies are enforced.

1. Bell–LaPadula Model

Focus: Confidentiality (Secrecy)

Common in military and government. It prevents data leaks.

  • No Read Up: A user cannot read data at a higher classification level.
  • No Write Down: A user cannot write sensitive data to lower levels.
Goal: Keep secrets secret.

2. Biba Model

Focus: Integrity (Trustworthiness)

Concerned with ensuring data is not corrupted or modified by unreliable sources.

  • No Write Up: Lower-level users cannot change high-level data.
  • No Read Down: Do not read potentially unreliable lower-level data.
Goal: Keep data accurate.

3. Clark–Wilson Model

Focus: Commercial Integrity

Best suited for banking and accounting. It relies on:

  • Well-formed transactions: Steps must be followed exactly.
  • Separation of Duties: Splitting tasks.
  • Auditing: Tracking who did what.

4. Access Control Models

  • DAC (Discretionary): The resource owner decides who gets in (e.g., social media privacy settings).
  • MAC (Mandatory): The system decides based on strict labels (Top Secret, Secret).
  • RBAC (Role-Based): Access is based on your job title. Most common in modern companies.
